## [0.5.0] - 2026-07-17

### 기능
- **터미널 Hex/Raw 스트림 진단 패널**: 선택적으로 세션 I/O를 바이트 단위로 표시해 인코딩/이스케이프 문제 진단
- **JSON 호스트 데이터 소스**: 로컬 JSON 또는 HTTP(S)에서 호스트 목록 가져오기(CMDB / Ansible / 커스텀 API); 메타데이터만, 비밀 거부; HTTP 인증 헤더
- **호스트 인벤토리 공유·가져오기**: 메타데이터만 내보내기(Ansible YAML 포함); 클립보드 가져오기
- **이름 있는 워크스페이스 템플릿**: 호스트 바인딩, 분할 레이아웃, 선택적 cwd/시작 명령을 저장 후 빠른 전환기에서 적용
- **연결 로그 북마크**: 재생 위치 북마크 + 메모 + 검색; 목록에 북마크 수 표시
- **포트 포워딩 라이브 채널 보기**: 로컬/원격/동적 포워드의 연결별 소스·대상·바이트
- **스크립트 onOutput 동작 확장**: 일치 시 데스크톱 알림, 소리, 탭 표시, 세션 녹화 시작
- **안전 붙여넣기·정밀 브로드캐스트**: 여러 줄 지연/프롬프트 대기/위험 명령 확인; 워크스페이스/선택/그룹/창 지정
- **시스템 OpenSSH 채널 강화**: GSSAPI/Kerberos 및 포스트퀀텀(PQ)을 시스템 OpenSSH로

### Windows ARM64
- **win-arm64에 mosh / ET 포함**: MoshMagies 0.1.9, EternalTerminal 6.2.10 네이티브 Windows arm64
- **win-arm64 전용 자동 업데이트**: `latest-arm64.yml` 채널(x64 추종 중단)

## [0.4.6] - 2026-07-15

### 보안
- **SSH 호스트 키 검증 해제가 더 이상 조용하지 않음**: `verifyHostKeys`가 꺼져 있으면(터미널 세션 및 Mosh 통계 연결) 프롬프트 없이 모든 호스트 키를 수락한다는 경고를 로그에 남김
- **설정 페이지 지속 경고**: 「SSH 호스트 키 검증」을 끄면 토글 아래 MITM 위험 안내가 계속 표시됨(en / zh-CN / zh-TW). 기본값은 켜짐

## [0.4.5] - 2026-07-15

### 수정
- **중첩 암호문으로 인한 401 / 빈 스트림**: 키체인 불가 상태에서 반복 저장 시 암호화가 중첩됨(`enc:v2(enc:v1(...))`); 복호화 루프 한계를 수정해 예산 내 중첩 블롭을 완전 해제
- **하나의 잘못된 자격 증명이 전체 vault 로드를 망가뜨리지 않음**: 필드 복호화 실패 시 저장값을 유지(fail-soft)해 vault는 로드되고 키체인 복구 후에도 키 복구 가능
- **웹 검색 API 키**: 복호화 실패 후 포커스/블러만으로 저장된 키를 삭제하지 않음; 암복호화 오류 메시지 명확화
- **Windows DPAPI 암호문 감지**: 재암호화 가드가 DPAPI 키(`AQAAAN` 헤더)를 놓치던 문제 수정
- **Cursor Agent**: 복호화 실패 시 암호문을 API 키로 자식 프로세스에 주입하지 않음
- 설정 Provider / 웹 검색 / Cursor: 복호화 실패 시 키 재입력 안내 통일; UI 언어 전환이 미저장 키를 덮어쓰지 않음

## [0.2.0] - 2026-07-11

### 기능
- 자동 업데이트 IPC를 모든 창(메인 + 설정)에 브로드캐스트
- 수동 확인과 자동 업데이트 상태 머신 통합
- 설정에서 「업데이트 확인」 시 다운로드 진행률 실시간 표시
- 시작 약 5초 후 electron-updater 자동 확인
- 새 버전 발견 시 자동 다운로드(`autoDownload=true`)
- 다운로드 완료 시 지속 토스트(「지금 다시 시작」으로 설치)
- 다운로드 실패 시 오류 토스트 및 Releases 폴백
- 설정 → 시스템 진행 바가 `useUpdateCheck`로 구동

### 설계 메모
- `broadcastToAllWindows`가 단일 발신자 IPC를 대체
- `manualCheckStatus`가 수동 확인 UI를 추적; `autoDownloadStatus`와 우선순위 렌더
- `SettingsSystemTab`은 `useUpdateCheck`의 순수 소비자
- 전역 IPC 리스너는 `autoUpdateBridge.init()`에서 한 번 등록
- `autoInstallOnAppQuit=false` — 사용자가 재시작을 시작

### SettingsSystemTabProps 인터페이스
- 제거: `autoDownloadStatus`, `downloadPercent`
- 추가: `updateState`, `checkNow`, `installUpdate`, `openReleasePage`

### 참고
- 패키지 앱(Windows NSIS, macOS dmg/zip, Linux AppImage)에 적용; 개발 시 `forceDevUpdateConfig=true` + `dev-app-update.yml` 필요
- 자동 다운로드 중 레거시 `hasUpdate` 토스트 억제

### CI / 빌드
- 서명 불가 시 무료 Linux 패키지 우선
- Linux x64(AlmaLinux 8): Clang 우선, gcc-toolset-13 폴백
- Linux arm64(Debian Bullseye): `clang-14 + lld-14`
- Release job이 Linux 산출물만으로 게시 가능
- 플랫폼 스킵 시 deb 검사를 warning으로 완화
