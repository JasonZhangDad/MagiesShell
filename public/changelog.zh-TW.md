## [0.5.0] - 2026-07-17

### 功能
- **終端 Hex/Raw 流診斷面板**：可選開啟，逐位元組查看工作階段原始輸入輸出，便於排查編碼/跳脫序列問題
- **JSON 主機資料源**：從本機 JSON 或 HTTP(S) 拉取主機清單（CMDB / Ansible / 自訂 API 風格），僅中繼資料、拒絕含金鑰清單；支援 HTTP 認證標頭
- **主機清單分享與匯入**：匯出僅含中繼資料的清單供團隊交接（含 Ansible YAML），剪貼簿匯入
- **命名工作區範本**：將主機綁定、分割版面、可選 cwd/啟動命令存成範本，快速切換器一鍵套用
- **連線日誌書籤**：回放位置書籤 + 備註 + 搜尋跳轉，日誌列表顯示書籤數
- **連接埠轉發即時通道檢視**：本機/遠端/動態轉發的逐連線來源、目標與流量位元組統計
- **腳本 onOutput 觸發動作擴充**：命中輸出模式可選桌面通知、提示音、標記分頁、開始工作階段錄製
- **安全貼上與精確廣播**：多行貼上延遲/等待提示符/危險命令確認；廣播可精確指定工作區/選定/分組/視窗
- **系統 OpenSSH 通道增強**：GSSAPI/Kerberos 與後量子（PQ）演算法經系統 OpenSSH 支援

### Windows ARM64
- **win-arm64 安裝包補齊 mosh / ET 捆綁**：MoshMagies 0.1.9 與 EternalTerminal 6.2.10 首發 Windows arm64 原生二進位
- **win-arm64 獨立自動更新源**：更新中繼資料改走 `latest-arm64.yml`，不再跟隨 x64 更新

## [0.4.10] - 2026-07-17

### 功能
- **SSH 連線診斷中心**：主機編輯「測試連線」+ 失敗時「執行診斷」，分步檢查 DNS / TCP / 跳板 / 主機金鑰 / 認證 / SFTP
- **SSH Agent 一等公民認證**：可顯式選擇 Agent、檢視指紋並指定偏好身分；連線日誌記錄實際認證方式
- **多主機健康快照**：Vault 一鍵批次檢查延遲、認證與負載/記憶體/磁碟
- **SFTP 可靠性一期**：斷點續傳、失敗自動退避重試、傳輸佇列持久化、可選 SHA-256 校驗
- **產品化引導**：首次空 Vault 三步引導；快速切換器命令項；空狀態遷移提示；首次連線成功建議

## [0.4.6] - 2026-07-15

- **SSH 主機金鑰校驗關閉時不再靜默**：`verifyHostKeys` 關閉時（終端工作階段與 mosh 統計連線）會記錄明確的告警日誌，說明正在不加詢問地接受任意主機金鑰

## [0.4.5] - 2026-07-15

### 修復
- **巢狀密文導致的 401 / 空流**：鑰匙圈失效期間反覆儲存會把金鑰層層加密（`enc:v2(enc:v1(...))`）；解密迴圈邊界修正後，預算內的多層巢狀能完整解出
- **單個壞憑證不再拖垮整個憑證庫載入**：欄位解密失敗時改為保留原儲存值（fail-soft），憑證庫照常載入
- **Web 搜尋 API Key**：解密失敗後僅聚焦/失焦不再誤刪已存金鑰；補充解密/加密失敗的明確提示
- **Windows DPAPI 密文識別修正**：防重複加密守衛此前漏判 DPAPI 金鑰（`AQAAAN` 頭）
- **Cursor Agent**：解密失敗時不再把密文當 API Key 注入子行程
- 設定頁 Provider / Web 搜尋 / Cursor 三處統一：解密失敗時明確提示重新輸入 Key，切換介面語言不再覆蓋未儲存的 Key

## [0.2.0] - 2026-07-11

### 功能
- 修復自動更新 IPC 事件僅傳送到單個視窗的問題，改為廣播所有視窗（主視窗 + 設定視窗均可收到）
- 統一手動檢查更新與自動更新的狀態機，消除三套並行狀態
- 設定視窗中點擊「檢查更新」後，下載進度可即時反映在 UI 中
- 應用啟動後 5 秒自動觸發 `electron-updater` 檢查更新
- 發現新版本後自動開始下載（`autoDownload=true`）
- 下載完成後彈出持久 toast 通知，使用者點擊「立即重啟」即可安裝
- 下載失敗時彈出錯誤 toast，提供「開啟 Releases」降級入口
- Settings > System 進度條即時展示自動下載進度，由 `useUpdateCheck` 統一驅動

### 設計原理
- `broadcastToAllWindows` 替換 `getSenderWindow` 單點傳送
- `manualCheckStatus` 欄位追蹤手動檢查 UI 狀態，與 `autoDownloadStatus` 在 UI 層按優先級渲染
- `SettingsSystemTab` 不再持有本機 update state，單向接收 `useUpdateCheck` 統一資料
- 全域持久化 IPC 監聽器在 `autoUpdateBridge.init()` 時一次性註冊
- `autoInstallOnAppQuit=false`，不做靜默安裝，由使用者主動觸發重啟

### 介面變更（SettingsSystemTabProps）
- 移除：`autoDownloadStatus`、`downloadPercent`
- 新增：`updateState`、`checkNow`、`installUpdate`、`openReleasePage`

### 注意事項
- 此功能僅對打包後的應用（Windows NSIS、macOS dmg/zip、Linux AppImage）生效
- `hasUpdate` 舊 toast 在 `autoDownloadStatus !== 'idle'` 時自動抑制

### CI / 建置改進
- 跳過 macOS / Windows 建置（需要付費程式碼簽章憑證）時，專注提供免費 Linux 發行包
- Linux x64（AlmaLinux 8）編譯器升級：優先使用 Clang，回退 gcc-toolset-13
- Linux arm64（Debian Bullseye）編譯器升級：`clang-14 + lld-14`
- Release job 可僅基於 Linux 產物發布
- 軟化 deb 產物校驗：找不到檔案時輸出 warning 而非 error
