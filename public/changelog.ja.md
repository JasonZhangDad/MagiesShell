## [0.5.0] - 2026-07-17

### 機能
- **端末 Hex/Raw ストリーム診断パネル**：任意でセッション入出力をバイト単位表示し、符号化/エスケープ列の調査を支援
- **JSON ホストデータソース**：ローカル JSON または HTTP(S) からホスト一覧を取得（CMDB / Ansible / カスタム API 風）；メタデータのみ、秘密を拒否；HTTP 認証ヘッダ対応
- **ホスト一覧の共有とインポート**：メタデータのみの一覧をエクスポート（Ansible YAML 含む）；クリップボードからインポート
- **名前付きワークスペーステンプレート**：ホスト割当、分割レイアウト、任意の cwd/起動コマンドを保存しクイックスイッチャーで適用
- **接続ログのブックマーク**：再生位置のブックマーク + メモ + 検索；ログ一覧に件数表示
- **ポート転送のライブチャネル表示**：ローカル/リモート/ダイナミック転送の接続ごとの送信元・先・バイト数
- **スクリプト onOutput アクション拡張**：一致時にデスクトップ通知、音、タブマーク、セッション録画開始
- **安全な貼り付けと精密ブロードキャスト**：複数行貼り付け遅延/プロンプト待ち/危険コマンド確認；ワークスペース/選択/グループ/ウィンドウ指定
- **システム OpenSSH チャネル強化**：GSSAPI/Kerberos と後量子（PQ）をシステム OpenSSH 経由で

### Windows ARM64
- **win-arm64 に mosh / ET 同梱**：MoshMagies 0.1.9 と EternalTerminal 6.2.10 の Windows arm64 ネイティブ
- **win-arm64 専用自動更新**：`latest-arm64.yml` チャンネル（x64 追従を廃止）

## [0.4.6] - 2026-07-15

### セキュリティ
- **SSH ホスト鍵検証オフが無言でなくなった**：`verifyHostKeys` 無効時（端末セッションと Mosh 統計接続）は、プロンプトなしで任意のホスト鍵を受け入れる旨を明確にログ
- **設定ページの常時警告**：「SSH ホスト鍵を検証」をオフにするとトグル下に MITM リスク案内を表示（en / zh-CN / zh-TW）。既定はオン

## [0.4.5] - 2026-07-15

### 修正
- **入れ子暗号文による 401 / 空ストリーム**：キーチェーン不可時の繰り返し保存で暗号化が入れ子に（`enc:v2(enc:v1(...))`）；復号ループ境界を修正し予算内で完全に解凍
- **1 件の壊れた資格情報が vault 全体の読み込みを壊さない**：フィールド復号失敗時は保存値を保持（fail-soft）
- **Web 検索 API キー**：復号失敗後のフォーカス/ブラーだけで保存キーを消さない；明瞭なエラー
- **Windows DPAPI 暗号文検出**：再暗号化ガードが DPAPI キー（`AQAAAN` ヘッダ）を見逃していた問題を修正
- **Cursor Agent**：復号失敗時に暗号文を API キーとして子プロセスに注入しない
- 設定の Provider / Web 検索 / Cursor：復号失敗時の再入力案内を統一；UI 言語切替が未保存キーを上書きしない

## [0.4.4] - 2026-07-14

## [0.2.0] - 2026-07-11

### 機能
- 自動更新 IPC を全ウィンドウ（メイン + 設定）へブロードキャスト
- 手動チェックと自動更新の状態機械を統一
- 設定の「更新を確認」でダウンロード進捗をリアルタイム表示
- 起動約 5 秒後に electron-updater で自動チェック
- 新バージョン検出時に自動ダウンロード（`autoDownload=true`）
- 完了時の持続トースト（「今すぐ再起動」でインストール）
- 失敗時エラー toast と Releases フォールバック
- 設定 → システムの進捗バーは `useUpdateCheck` 駆動

### 設計メモ
- `broadcastToAllWindows` が単一送信 IPC を置換
- `manualCheckStatus` が手動チェック UI を追跡；`autoDownloadStatus` と優先度レンダ
- `SettingsSystemTab` は `useUpdateCheck` の純粋な消費者
- グローバル IPC リスナは `autoUpdateBridge.init()` で一度登録
- `autoInstallOnAppQuit=false` — ユーザーが再起動を開始

### SettingsSystemTabProps インターフェース
- 削除：`autoDownloadStatus`、`downloadPercent`
- 追加：`updateState`、`checkNow`、`installUpdate`、`openReleasePage`

### 注意
- パッケージアプリ（Windows NSIS、macOS dmg/zip、Linux AppImage）に適用；開発時は `forceDevUpdateConfig=true` + `dev-app-update.yml`
- 自動ダウンロード中はレガシー `hasUpdate` toast を抑制

### CI / ビルド
- 署名不可時は無料 Linux パッケージを優先
- Linux x64（AlmaLinux 8）：Clang 優先、gcc-toolset-13 フォールバック
- Linux arm64（Debian Bullseye）：`clang-14 + lld-14`
- Release job が Linux 成果物のみで公開可能
- プラットフォームスキップ時 deb 検査を warning に緩和
