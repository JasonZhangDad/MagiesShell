# История изменений

## [0.5.1] - 2026-07-18

### Релиз
- Установщики и метаданные автообновления линии 0.5.x (MgTerminal-releases)

## [0.5.0] - 2026-07-17

### Функции
- **Панель диагностики Hex/Raw терминала**: опциональный побайтовый просмотр I/O сессии
- **JSON-источник хостов**: инвентари из локального JSON или HTTP(S); только метаданные; HTTP auth-заголовки
- **Обмен и импорт инвентаря**: экспорт только метаданных (включая Ansible YAML); импорт из буфера
- **Именованные шаблоны рабочих мест**: привязки хостов, сплиты, опциональные cwd/стартовая команда
- **Закладки журнала подключений**: позиции + заметки + поиск
- **Живой вид каналов проброса портов**: источник, цель и байты на соединение
- **Расширения onOutput скриптов**: уведомление, звук, метка вкладки, запись сессии
- **Безопасная вставка и точный broadcast**: задержка / ожидание prompt / подтверждение опасных команд
- **Канал системного OpenSSH**: GSSAPI/Kerberos и post-quantum (PQ)

### Windows ARM64
- **win-arm64 с mosh / ET**: нативные бинарники MoshMagies 0.1.9 и EternalTerminal 6.2.10
- **Отдельный канал автообновления**: `latest-arm64.yml`

## [0.4.10] - 2026-07-17

- **Снимок здоровья нескольких хостов**: задержка, auth, load/mem/disk
- **Надёжность SFTP этап 1**: resume, auto-retry, постоянная очередь, опциональный SHA-256
- **Онбординг продукта**: пустой Vault, палитра команд, подсказки пустых состояний

### Исправления
- Существующие Vault без first-run; jump корректно закрывается при сбое auth health

## [0.4.6] - 2026-07-15

### Безопасность
- **Отключение проверки SSH host key больше не «тихое»**: при выключенном `verifyHostKeys` (терминальные сессии и Mosh stats) в лог пишется явное предупреждение, что любой host key принимается без запроса
- **Постоянное предупреждение в настройках**: выключение «Проверять SSH host keys» показывает риск MITM под переключателем (en / zh-CN / zh-TW). По умолчанию включено

## [0.4.5] - 2026-07-15

### Исправления
- **Вложенный шифротекст вызывал 401 / пустые потоки**: повторные сохранения без keychain вкладывали шифрование (`enc:v2(enc:v1(...))`); цикл расшифровки полностью разворачивает вложенные blob в пределах бюджета
- **Одна плохая учётная запись больше не ломает весь vault**: при сбое расшифровки поля сохраняется значение (fail-soft)
- **API-ключ веб-поиска**: только focus/blur после сбоя расшифровки больше не удаляют сохранённый ключ; более ясные сообщения
- **Распознавание DPAPI на Windows**: защита от повторного шифрования пропускала DPAPI-ключи (заголовок `AQAAAN`) — исправлено
- **Cursor Agent**: при сбое расшифровки шифротекст не передаётся как API-ключ в дочерний процесс
- Настройки Provider / веб-поиск / Cursor: единые подсказки ввести ключ заново; смена языка UI не затирает несохранённые ключи

## [0.4.4] - 2026-07-14

### Исправления
- **ИИ 401 / пустые потоки**: при сбое расшифровки API-ключа или отсутствии синхронизации с main-процессом запросы не идут с плейсхолдером `__IPC_SECURED__`; немедленный сбой и подсказка сохранить ключ
- Перед отправкой ждать синхронизации providers с main-процессом
- Явные auth-ошибки, если локальный ключ недоступен

## [0.4.3] - 2026-07-14

### Исправления

- **Плейсхолдеры учётных данных**: границы подключения / облачной синхронизации распознают `enc:v2`

## [0.2.0] - 2026-07-11

### Функции
- IPC автообновления рассылается во все окна (главное + настройки)
- Единый state machine для ручной проверки и автообновления
- «Проверить обновления» в настройках показывает прогресс загрузки вживую
- Автопроверка через electron-updater ~через 5 с после запуска
- Автозагрузка при новой версии (`autoDownload=true`)
- Постоянный toast по завершении («Перезапустить сейчас»)
- Toast ошибки при сбое загрузки с fallback на Releases
- Прогресс-бар Настройки → Система управляется `useUpdateCheck`

### Заметки по дизайну
- `broadcastToAllWindows` заменяет IPC одному отправителю
- `manualCheckStatus` отслеживает UI ручной проверки; приоритет с `autoDownloadStatus`
- `SettingsSystemTab` — чистый consumer `useUpdateCheck`
- Глобальные IPC-слушатели регистрируются один раз в `autoUpdateBridge.init()`
- `autoInstallOnAppQuit=false` — перезапуск инициирует пользователь

### Интерфейс SettingsSystemTabProps
- Удалено: `autoDownloadStatus`, `downloadPercent`
- Добавлено: `updateState`, `checkNow`, `installUpdate`, `openReleasePage`

### Примечания
- Для упакованных приложений (Windows NSIS, macOS dmg/zip, Linux AppImage); в dev: `forceDevUpdateConfig=true` + `dev-app-update.yml`
- Устаревший toast `hasUpdate` подавляется во время автозагрузки

### CI / сборка
- Предпочитать бесплатные Linux-пакеты, если нет подписи
- Linux x64 (AlmaLinux 8): Clang предпочтителен, fallback gcc-toolset-13
- Linux arm64 (Debian Bullseye): `clang-14 + lld-14`
- Release job может публиковать только из Linux-артефактов
- Смягчить проверки deb до warning, если платформа пропущена
